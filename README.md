# 程序员修炼之道：通向务实的最高境界

```
ETC(earsier to change)
RDY（重复性）
正交性
契约式设计
面向未来编程，确实很难：薛定谔的猫
父类实现接口细节的继承有继承税 -> 委托：父类继承接口，子类继承父类则需要实现接口，避免了继承税
处理并发问题：信号量、锁、事务、角色模型？
务实的程序员连自己都不相信
直觉==蜥蜴脑，当某些现象出现时，蜥蜴脑会告诉你一些信息
测试是代码的第一个用户
程序入侵可能不是攻击者多厉害，而是程序员太粗心了
禁用密码框的复制沾沾功能，会使用户设置更容易破解的密码
糟糕的设计、错误的决定、低劣的代码，不去处理，软件熵越积越多，技术债就越来越多
```
